# Neo4j JDBC Bearer Token Test Client

Java client that acquires an OAuth 2.0 token from Keycloak via client credentials grant and connects to Neo4j Enterprise Edition using the [Neo4j JDBC driver](https://github.com/neo4j/neo4j-jdbc) with bearer authentication.

## Prerequisites

- Java 21+
- A deployed Neo4j instance with OIDC configured (see [deployment instructions](../README.md#keycloak-setup))
- A running Keycloak instance (Azure Container Apps or local Docker)

## Quick Start

The fastest path when both Keycloak and Neo4j are already deployed via the project CLI:

```bash
cd oauth-java
./run.sh
```

`run.sh` reads the most recent deployment from `../.deployments/`, generates a `.env` with the Keycloak and Neo4j connection details, builds the project, and runs the test. You can also target a specific scenario:

```bash
./run.sh standalone-v2025
```

## Running Manually

### Against Azure deployments (Keycloak only, no Neo4j connection)

```bash
./gradlew run --args="--azure"
```

Reads Keycloak config from `../keycloak-infra/.deployment.json`, acquires a token, and displays the decoded JWT claims. Useful for verifying the token structure before Neo4j is deployed.

### Against local Keycloak

```bash
./gradlew run
```

Connects to `http://localhost:8080` with the default realm (`neo4j`) and client credentials (`neo4j-client` / `neo4j-client-secret`).

### With explicit arguments

```bash
./gradlew run --args="--server-url=https://keycloak.example.com --neo4j-uri=bolt://neo4j.example.com:7687"
```

## Configuration

The client resolves configuration in this order:

1. CLI arguments (`--server-url`, `--client-id`, `--client-secret`, `--realm`, `--neo4j-uri`)
2. `.env` file (generated by `run.sh`)
3. `--azure` flag (reads `../keycloak-infra/.deployment.json`)
4. Built-in defaults (localhost Keycloak, no Neo4j connection)

### .env format

```
NEO4J_URI=bolt://vm0.neo4j-xxx.eastus2.cloudapp.azure.com:7687
KEYCLOAK_URL=https://keycloak-xxx.azurecontainerapps.io
TOKEN_ENDPOINT=https://keycloak-xxx.azurecontainerapps.io/realms/neo4j/protocol/openid-connect/token
CLIENT_ID=neo4j-client
CLIENT_SECRET=75f93e4f-...
REALM=neo4j
```

## What It Does

1. Hits the Keycloak OIDC discovery endpoint to verify reachability
2. Acquires an access token via OAuth 2.0 client credentials grant
3. Decodes the JWT and displays all claims, highlighting the fields Neo4j uses (`aud`, `roles`, `sub`)
4. If `NEO4J_URI` is configured, connects to Neo4j via JDBC with `authScheme=bearer`
5. Runs `SHOW CURRENT USER` to display the OIDC-mapped user and roles
6. Runs a test Cypher query to confirm read access

## How Bearer Auth Works with the JDBC Driver

The Neo4j JDBC driver supports bearer token authentication through connection properties:

```java
var props = new Properties();
props.put("authScheme", "bearer");
props.put("password", accessToken);  // token goes in the password field

var conn = DriverManager.getConnection("jdbc:neo4j://host:7687", props);
```

The driver converts `bolt://` URIs to `jdbc:neo4j://` internally. When `authScheme` is set to `bearer`, the driver sends the token value from the `password` property as a Bolt bearer auth token. The `user` property is ignored.
